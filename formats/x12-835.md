# X12 835 Remittance Format

## Trigger Phrases

- 835
- X12 835
- remittance
- ERA
- electronic remittance
- payment advice
- EOB
- explanation of benefits

## Overview

X12 835 is the HIPAA-mandated standard for electronic remittance advice (ERA). This skill transforms HealthSim adjudication data into compliant 835 transactions for payment posting.

## Transaction Structure

```
ISA - Interchange Control Header
  GS - Functional Group Header
    ST - Transaction Set Header (835)
      BPR - Financial Information
      TRN - Reassociation Trace Number
      DTM - Production Date (optional)
      N1 Loop - Payer Identification
      N1 Loop - Payee Identification
      LX Loop - Header Number
        CLP Loop - Claim Payment Information
          NM1 - Patient Name
          SVC - Service Payment Information
          CAS - Claim Adjustment
          DTM - Service Dates
    SE - Transaction Set Trailer
  GE - Functional Group Trailer
IEA - Interchange Control Trailer
```

## Key Segments

### BPR - Financial Information
```
BPR*I*1250.00*C*ACH*CTX*01*123456789*DA*9876543210*1234567890**01*987654321*DA*1234567890*20250120~
```

| Element | Position | Description | Example |
|---------|----------|-------------|---------|
| BPR01 | Transaction Handling | I=Remittance Only, C=Payment | `I` |
| BPR02 | Total Payment Amount | Sum of all payments | `1250.00` |
| BPR03 | Credit/Debit | C=Credit, D=Debit | `C` |
| BPR04 | Payment Method | ACH, CHK, NON | `ACH` |
| BPR05 | Payment Format | CTX, CCD | `CTX` |
| BPR06-10 | Sender Bank Info | DFI ID, Account | Routing/Account |
| BPR12-16 | Receiver Bank Info | DFI ID, Account | Routing/Account |
| BPR16 | Check/Payment Date | CCYYMMDD | `20250120` |

### Payment Method Codes (BPR04)
| Code | Description |
|------|-------------|
| ACH | Automated Clearing House |
| CHK | Check |
| FWT | Federal Reserve Wire |
| NON | Non-Payment (zero-pay remit) |

### TRN - Reassociation Trace Number
```
TRN*1*TRACE123456*1234567890~
```

| Element | Description |
|---------|-------------|
| TRN01 | Trace Type (1=Current) |
| TRN02 | Reference ID |
| TRN03 | Originator ID |

### CLP - Claim Payment Information
```
CLP*CLM20250115001*1*175.00*125.00**12*AUTH001*11*1~
```

| Element | Position | Description | Example |
|---------|----------|-------------|---------|
| CLP01 | Patient Control Number | Claim ID | `CLM20250115001` |
| CLP02 | Claim Status | 1=Processed Primary | `1` |
| CLP03 | Total Claim Charge | Billed amount | `175.00` |
| CLP04 | Total Claim Payment | Paid amount | `125.00` |
| CLP05 | Patient Responsibility | Member owes | `50.00` |
| CLP06 | Claim Filing Indicator | 12=PPO, 16=HMO | `12` |
| CLP07 | Payer Claim Control # | Authorization | `AUTH001` |
| CLP08 | Facility Type | Place of service | `11` |
| CLP09 | Claim Frequency | 1=Original | `1` |

### Claim Status Codes (CLP02)
| Code | Description |
|------|-------------|
| 1 | Processed as Primary |
| 2 | Processed as Secondary |
| 3 | Processed as Tertiary |
| 4 | Denied |
| 19 | Processed as Primary, Forwarded to Additional Payer |
| 20 | Processed as Secondary, Forwarded to Additional Payer |
| 22 | Reversal of Previous Payment |
| 23 | Not Our Claim, Forwarded |

### SVC - Service Payment Information
```
SVC*HC:99214*175.00*125.00**1~
```

| Element | Position | Description | Example |
|---------|----------|-------------|---------|
| SVC01 | Procedure Code | Qualifier:Code:Mod | `HC:99214` |
| SVC02 | Line Charge | Billed amount | `175.00` |
| SVC03 | Line Payment | Paid amount | `125.00` |
| SVC04 | Revenue Code | For institutional | `0120` |
| SVC05 | Units | Quantity paid | `1` |
| SVC06 | Original Procedure | If changed | |

### CAS - Claim Adjustment
```
CAS*CO*45*50.00~
CAS*PR*1*25.00*2*10.00~
```

| Element | Description |
|---------|-------------|
| CAS01 | Claim Adjustment Group |
| CAS02 | Adjustment Reason Code (CARC) |
| CAS03 | Adjustment Amount |
| CAS04-CAS19 | Additional reason/amount pairs |

### Claim Adjustment Group Codes (CAS01)
| Code | Description | Financial Responsibility |
|------|-------------|--------------------------|
| CO | Contractual Obligations | Provider write-off |
| CR | Corrections and Reversals | Correction |
| OA | Other Adjustments | Various |
| PI | Payer Initiated Reductions | Plan reduction |
| PR | Patient Responsibility | Member pays |

| Code | Description | Group |

| 4 | Procedure code inconsistent with modifier | CO |
| 16 | Claim lacks information | CO |
| 18 | Duplicate claim/service | CO |
| 22 | Coordination of benefits | OA |
| 23 | Payment adjusted - prior payment | OA |
| 29 | Timely filing | CO |
| 45 | Charge exceeds fee schedule | CO |
| 50 | Non-covered services | CO |
| 96 | Non-covered charge(s) | CO |
| 97 | Benefit included in another service | CO |

## Validation Rules

1. **BPR02 must equal sum**: Total of all CLP04 amounts
2. **CLP amounts**: CLP03 >= CLP04 + CLP05 adjustments
3. **CAS amounts**: Must equal CLP03 - CLP04 - CLP05
4. **Segment counts**: SE01 must match actual count
5. **Control number matching**: ISA/IEA, GS/GE, ST/SE
